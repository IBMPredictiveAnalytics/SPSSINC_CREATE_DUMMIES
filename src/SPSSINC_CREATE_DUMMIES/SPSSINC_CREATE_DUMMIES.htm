Create a Set of Dummy Variables from Values of a Variable

This procedure accepts a list of variables and produces a set of dummy variables representing their values. It can produce dummies for individual variables, dummies for two-way interaction terms, and/or dummies for three-way interactions. It passes the data ignoring any cases excluded by the current filter and creates dummies for the values found.

For scale variables, by default, it produces just one variable instead of dummies which can be using in producing interaction terms.

Create Dummy Variables for. Choose variables from which to create sets of dummy variables. Typically these would be categorical variables, but scale variables can be usefully selected for building interaction terms. Both string and numeric variables can be used, but date and time variables are not allowed.

Main Effect Dummy Variables. Check the box Create main-effect dummies to create a set of dummy variables for each selected variable.

Root Names. Specify names to be used as the prefix of all the dummy variables for each variable selected. Enter as many names are there are variables selected, separated by a blank. The dummy names will have the form rootname_n. If a variable with that name already exists, it is replaced.

Macro Name. If names are entered in this field, a macro will be created for each set of dummy variables or scale variable. Enter as many names are there are variables selected if macro generation is required. It is conventional to start each macro name with "!".

Two-Way Interactions and Three-Way Interactions. Follow the same procedure as for Main Effect Dummy Variables, but enter only a single root name and macro name (if desired). Dummy names will have the form
rootname_m_n,
and existing variables will be replaced.

Dummy Variable Labels By default, labels for the dummy variables are constructed from the input variable name and the appropriate value label. Select Use values to use the values instead. If a value is not labelled, the value itself is used in the label.

Value Order. Choose Ascending or Descending for the sort order of the dummy variables. This is mainly useful in connection with generating a macro.

Omit first dummy category from macro definitions. Check this box to omit the first dummy variable from each macro definition. This is convenient for the REGRESSION procedure in order to avoid having a singular regressor matrix. If there are no interaction terms being generated, this means that the first dummy for each variable is omitted from the macro definition. If there are interaction terms, variables involving that category for any of the input variables are omitted from the macro definition.

• Which categories are considered as first depends on the Value Order setting.

• Scale variables are not affected by this setting.

Measurement Level Usage. Categorical variables produce a dummy for each distinct values. Scale variables are represented by a single variable. Choose Do not create dummies for scale variable values for this behavior. Choose Create dummies for all variables to ignore the measurement level and generate a variable for each value even for scale variables.

Additional Features

This dialog generates syntax for the SPSSINC CREATE DUMMIES extension command. To display help for this command, run the following syntax:

SPSSINC CREATE DUMMIES /HELP.

Requirements

This dialog requires the Integration Plug-In for Python and the SPSSINC CREATE DUMMIES extension command. For IBM® SPSS® Statistics 19 and higher, the Plug-In and the extension command are installed with the Essentials for Python package. For more information, see How to Get Integration Plug-Ins, under Core System > Frequently Asked Questions in the IBM SPSS Statistics Help system. If you downloaded SPSSINC CREATE DUMMIES from the SPSS Community, then please follow the instructions in the associated readme file.
