Создать набор фиктивных из значений некоторой переменной

Эта процедура принимает список переменных и выдает набор фиктивных переменных, представляющих их значения. Она может выдавать фиктивные переменные для отдельных переменных, для двусторонних взаимодействий и/или для трехсторонних взаимодействий. Она передает данные, игнорируя все наблюдения, исключенные текущим фильтром, и создает фиктивные переменные для найденных значений.

Для количественных переменных по умолчанию вместо фиктивных переменных генерируется только одна переменная, которую можно использовать для генерирования членов взаимодействия.

Создать фиктивные переменные для. Выберите переменные, из которых надо создать набор фиктивных переменных. Обычно это будут категорийные переменные, однако количественные переменные можно с пользой применять для построения членов взаимодействия. Можно использовать как текстовые, так и числовые переменные, но переменные даты и времени недопустимы.

Фиктивные переменные главного эффекта. Включите переключатель Создать фиктивные переменные главного эффекта, чтобы создать набор фиктивных переменных для каждой выбранной переменной.

Основы имен. Задайте имена, которые будут использоваться как префикс всех фиктивных переменных для каждой выбранной переменной. Введите столько имен, сколько выбрано переменных, разделяя их пробелами. Имена фиктивных переменных имеют вид основаимен_n. Если переменная с таким именем уже существует, она будет перезаписана.

Имя макроопределения. Если в этом поле введены имена, для каждого набора фиктивных переменных или количественной переменной будет создана макрокоманда. Если требуется сгенерировать макрокоманды, введите столько имен, сколько выбрано переменных. По договоренности имена макрокоманд начинаются с "!".

Двусторонние и трехсторонние взаимодействия. Испольуйте ту же процедуру, что и для фиктивных переменных главного эффекта, но вводите только одну основу имен и имя макрокоманды (если требуется). Имена фиктивных переменных имеют вид
основаимен_m_n,
а существующие переменные будут заменены.

Метки фиктивных переменных По умолчанию метки для фиктивных переменных строятся из имени входной переменной и метки соответствующего значения. Выберите Использовать значения, чтобы использовать вместо этого значения. Если значение не имеет метки, то в метке используется само значение.

Порядок значений. Выберите Восходящий или Нисходящий порядок сортировки фиктивных переменных. Это, главным образом, полезно в связи с созданием макроопределений.

Не включать первую фиктивную категорию из определений макрокоманд. Включите этот переключатель, чтобы пропустить первую фиктивную категорию из каждого определения макрокоманды. Это удобно для процедуры REGRESSION, чтобы избежать получения вырожденной матрицы регрессии. Если члены взаимодействия не генерируются, это означает, что первая фиктивная переменная для каждой переменной опускается в определении макрокоманды. Если члены взаимодействия присутствуют, переменные, включающие эту категорию для любой из входных переменных, опускаются в определении макрокоманды.

• Какие категории рассматриваются как первые, зависит от параметра Порядок значений.

• На количественные переменные этот параметр не влияет.

Использование уровня измерений. Категорийные переменные порождают фиктивную переменную для каждого отдельного значения. Количественные переменные представляются одной переменной. Выберите Не создавать фиктивные переменные для значений количественных переменных для такого поведения. Выберите Создавать фиктивные переменные для всех переменных, чтобы игнорировать тип измерений и создавать переменные для каждого значения любой переменной, в том числе количественной.

Дополнительные возможности

Это диалоговое окно формирует синтаксис для команды расширения SPSSINC CREATE DUMMIES. Чтобы показать справку для этой команды, выполните следующий синтаксис:

SPSSINC CREATE DUMMIES /HELP.
